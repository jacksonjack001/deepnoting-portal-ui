Agent 监控客户端配置

把 Codex / Claude Code 的 hook 事件上报到AI Proxy门户后，“当前 Agent 任务”页面会展示会话首个问题、当前状态、工具步骤、模型、token 和上下文窗口。

上报地址https://example.com/portal/api/agent/events
认证方式Authorization: Bearer YOUR_KEY，使用门户分配的 API Key。
显示位置登录门户后进入左侧“当前 Agent 任务”。

1. 准备环境变量

建议把 Key 放进用户级环境变量。下面的 YOUR_KEY 替换为你的AI Proxy API Key。

# macOS / Linux / WSL
export AI_PROXY_API_KEY="YOUR_KEY"
export AI_PROXY_AGENT_ENDPOINT="https://example.com/portal/api/agent/events"
export CODEX_CONTEXT_WINDOW="1050000"
# Windows PowerShell，设置后重新打开终端
[Environment]::SetEnvironmentVariable("AI_PROXY_API_KEY", "YOUR_KEY", "User")
[Environment]::SetEnvironmentVariable("AI_PROXY_AGENT_ENDPOINT", "https://example.com/portal/api/agent/events", "User")
[Environment]::SetEnvironmentVariable("CODEX_CONTEXT_WINDOW", "1050000", "User")

2. 先测试上报接口

这一步用于确认 Key、网络和门户服务都正常。执行成功后，看板里会出现一条 manual-test 测试会话。

# macOS / Linux / WSL
curl -fsS -X POST https://example.com/portal/api/agent/events \

  -H "Content-Type: application/json" \
  -d '{
    "client": "manual-test",

    "model": "gpt-5.5",

$body = @{
  client = "manual-test"

} | ConvertTo-Json
Invoke-RestMethod -Method Post "https://example.com/portal/api/agent/events" `
  -Headers @{ Authorization = "Bearer $env:AI_PROXY_API_KEY" } `
  -ContentType "application/json" `
  -Body $body

3. Claude Code 配置

Claude Code 可以直接使用 HTTP hook。用户级配置文件位置：macOS / Linux 为 ~/.claude/settings.json，Windows 为 %USERPROFILE%\.claude\settings.json。如果文件里已经有配置，请合并 hooks 字段，不要直接覆盖原文件。

# macOS / Linux / WSL
mkdir -p ~/.claude
nano ~/.claude/settings.json
# Windows PowerShell
New-Item -ItemType Directory -Force "$env:USERPROFILE\.claude"
notepad "$env:USERPROFILE\.claude\settings.json"

4. Codex 配置

Codex 的 hook 使用命令执行方式最稳。先放一个本地转发脚本，再在 hooks 配置里调用它。

创建转发脚本

脚本位置建议使用 ~/.codex/hooks/ai-proxy-agent-event.py；Windows 对应 %USERPROFILE%\.codex\hooks\ai-proxy-agent-event.py。

# macOS / Linux / WSL
mkdir -p ~/.codex/hooks
nano ~/.codex/hooks/ai-proxy-agent-event.py
chmod +x ~/.codex/hooks/ai-proxy-agent-event.py
# Windows PowerShell
New-Item -ItemType Directory -Force "$env:USERPROFILE\.codex\hooks"
notepad "$env:USERPROFILE\.codex\hooks\ai-proxy-agent-event.py"

ai-proxy-agent-event.py

#!/usr/bin/env python3
import json
import os
import sys
import urllib.request


endpoint = os.getenv("AI_PROXY_AGENT_ENDPOINT", "https://example.com/portal/api/agent/events")
api_key = os.getenv("AI_PROXY_API_KEY")

if not api_key:
    sys.exit(0)

try:
    payload = json.load(sys.stdin)
except Exception:
    payload = {}

payload.setdefault("client", "codex")

context_window = os.getenv("CODEX_CONTEXT_WINDOW")
if context_window and not payload.get("context_window"):
    try:
        payload["context_window"] = int(context_window)
    except ValueError:
        pass

used_tokens = payload.get("used_tokens") or payload.get("total_tokens") or payload.get("token_count")
if used_tokens is not None and payload.get("context_window") and not payload.get("remaining_context"):
    try:
        payload["remaining_context"] = max(int(payload["context_window"]) - int(used_tokens), 0)
    except (TypeError, ValueError):
        pass

body = json.dumps(payload).encode("utf-8")
request = urllib.request.Request(
    endpoint,
    data=body,
    headers={
        "Authorization": f"Bearer {api_key}",
        "Content-Type": "application/json",
        "X-Agent-Client": "codex",
    },
    method="POST",
)

try:
    urllib.request.urlopen(request, timeout=5).read()
except Exception as exc:
    print(f"ai-proxy agent monitor skipped: {exc}", file=sys.stderr)

sys.exit(0)

5. 看板字段说明

字段 来源 说明
第一个问题 prompt / first_prompt / user_prompt 同一会话只保留首次问题，便于看板识别任务。
当前状态 hook_event_name / event_type 根据开始、工具调用、压缩、停止等事件转换为运行中、工具执行中、已完成等状态。
已用 Token used_tokens / total_tokens / usage.total_tokens 客户端原始 hook 不一定提供 token；没有上报时显示 -。
剩余上下文 remaining_context 或 context_window - used_tokens 只上报窗口大小时会显示 未上报 / 窗口大小。

6. 常见问题

看板没有数据先执行第 2 步测试接口。如果测试成功但客户端无数据，检查 hook 文件路径、Key 环境变量和客户端是否重启。
只显示状态不显示 token这是正常情况。很多客户端 hook 默认不携带 token 统计；真实用量仍可在门户“总览与用量”查看。
429 tokens 限速CLAUDE_CODE_MAX_OUTPUT_TOKENS=128000 会要求 160K 或更高 TPM；30K TPM 档位请降低输出上限或升级 TPM。
避免阻塞编码HTTP 超时建议保持 5-10 秒；脚本失败时退出码保持 0，避免影响 Codex / Claude Code 主流程。
安全边界不要把真实 Key 写进项目仓库。hook payload 会在服务端做敏感字段脱敏后再保存。
