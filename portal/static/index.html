快速调用案例

支付激活后，把 API Key 放到 Authorization 头即可调用 OpenAI 兼容接口。

curl https://llm.example.com/v1/chat/completions \
  -H "Authorization: Bearer YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen3-32b",
    "messages": [
      {"role": "system", "content": "你是一个简洁可靠的助手。"},
      {"role": "user", "content": "用三句话介绍AI Proxy。"}
    ]
  }'

Python 示例

from openai import OpenAI

client = OpenAI(
    api_key="YOUR_KEY",
    base_url="https://llm.example.com"
)

resp = client.chat.completions.create(
    model="qwen3-32b",
    messages=[{"role": "user", "content": "你好"}],
)
print(resp.choices[0].message.content)

JavaScript 示例

import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.AI_PROXY_API_KEY,
  baseURL: "https://llm.example.com",
});

const resp = await client.chat.completions.create({
  model: "qwen3-32b",
  messages: [{ role: "user", content: "你好" }],
});
console.log(resp.choices[0].message.content);

Codex 配置

配置文件：macOS / Linux 为 ~/.codex/config.toml，Windows 为 %USERPROFILE%\.codex\config.toml。

安装说明

环境变量

# macOS / Linux / WSL
export AI_PROXY_API_KEY="YOUR_KEY"

# Windows PowerShell，设置后重新打开终端
[Environment]::SetEnvironmentVariable("AI_PROXY_API_KEY", "YOUR_KEY", "User")

config.toml

model_provider = "ai-proxy"
model = "gpt-5.5"
model_reasoning_effort = "medium"
plan_mode_reasoning_effort = "xhigh"
model_reasoning_summary = "none"
model_context_window = 1050000
model_auto_compact_token_limit = 945000
tool_output_token_limit = 6000
approval_policy = "never"
sandbox_mode = "danger-full-access"
suppress_unstable_features_warning = true
cache_ttl = "30m"

[model_providers.ai-proxy]
name = "OpenAI"
base_url = "https://llm.example.com"
wire_api = "responses"
requires_openai_auth = false
env_key = "AI_PROXY_API_KEY"
supports_websockets = false

[features]
responses_websockets_v2 = false
multi_agent = true
remote_connections = true

[agents]
max_threads = 4
max_depth = 1
job_max_runtime_seconds = 1800

[tui]
status_line = ["model-with-reasoning", "current-dir"]

[tui.model_availability_nux]
"gpt-5.5" = 4

Claude Code 配置

配置文件：macOS / Linux 为 ~/.claude/settings.json，Windows 为 %USERPROFILE%\.claude\settings.json。

安装说明

环境变量

# macOS / Linux / WSL
export AI_PROXY_API_KEY="YOUR_KEY"
export ANTHROPIC_BASE_URL="https://example.com/portal/anthropic"
export ANTHROPIC_AUTH_TOKEN="$AI_PROXY_API_KEY"

# Windows PowerShell，设置后重新打开终端
[Environment]::SetEnvironmentVariable("AI_PROXY_API_KEY", "YOUR_KEY", "User")
[Environment]::SetEnvironmentVariable("ANTHROPIC_BASE_URL", "https://example.com/portal/anthropic", "User")
[Environment]::SetEnvironmentVariable("ANTHROPIC_AUTH_TOKEN", "YOUR_KEY", "User")

更多调用文档举例
服务端保存 Key不要把 Key 写进前端代码或移动端包，统一由服务端转发调用。
设置超时和重试网络调用建议 30-60 秒超时，只对 429/5xx 做有限重试。
控制上下文长对话要摘要历史消息，避免无意义 token 消耗。
按模型分层轻量任务使用低成本模型，复杂推理再切换高阶模型。
记录 request_id线上排障时保留请求时间、模型名和 request_id。
监控余额接近预算上限时及时续费，避免业务调用突然被阻断。
